# 习题册数据标注工具 — 项目交接文档

## 一、项目概述

一个**习题册数据标注工作台**，让 13 位标注员在不同浏览器上同时对扫描的数学练习册进行标注（框选题干位置、标记答案、判断对错），标注数据通过后端 API 实时共享。

| 项目项 | 信息 |
|--------|------|
| GitHub 仓库 | https://github.com/mozhuyouchi/panlong-data-annotation-tool |
| 在线地址 | http://47.98.108.88/ |
| 服务器 | 阿里云 ECS（Ubuntu，1.6G 内存，40G 磁盘） |

## 二、技术架构

```
浏览器 ──→ Nginx (端口80)
              ├── /           → 静态网页 (/var/www/html/)
              ├── /data/      → 扫描图片 (/var/www/html/data/, 22GB)
              └── /api/*      → Flask API (127.0.0.1:5000) → SQLite
```

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | 纯 HTML/CSS/JS | 单页面应用，无框架 |
| 后端 API | Python Flask | 约 100 行代码 |
| 数据库 | SQLite | 只存标注数据，不存图片和配置 |
| 反向代理 | Nginx | 同域名避免 CORS 问题 |

## 五、标注工作流程

### 1. 准备工作
- 打开 http://47.98.108.88/
- 首次进入按 **Ctrl+F5 强制刷新**（只需一次）
- 导入最新的 `project-config.json`

### 2. 选择标注员和任务
- 左侧下拉选择标注员（001~013）
- 点击「进入我的任务」
- 系统自动加载服务器上的扫描图片

### 3. 开始标注

#### 模板页
1. 选择题目类型（单选题/多选题/填空题/问答题）
2. 在图片上**框选题干区域** → 再**框选作答区域**
3. 输入标准答案
4. 点右箭头进入下一页

#### 副本页
- 题干框和作答框已自动复制（从模板）
- 学生答案默认 = 标准答案，状态默认 = 正确
- **只需核查**，有错题才改状态

### 4. 标注框拖拽和缩放
- **移动**：鼠标直接拖拽标注框
- **缩放**：悬停在框上，四角出现白色方块，拖拽调整
- 绘制新框时不会误触旧框（自动屏蔽拖拽）

### 5. 页码对齐（如有需要）
- 页码对齐工具栏在左侧「我的任务」下方
- 手动输入实际逻辑页码 → 确认并重排
- 无效页点「当前扫描图无效」

### 6. 导出结果
- 完成标注后点击「导出任务 JSON」
- 页码校正完成后点击「导出修正后配置」

## 六、数据存储说明

| 数据 | 存储位置 | 同步方式 |
|------|---------|---------|
| 页码校正 | **本地文件**（JSON） | 手动导入导出，不经服务器 |
| 标注内容（templates + attempts） | **云端**（SQLite） | 自动推送/拉取 |

不同标注员的标注数据有独立 key，不会互相覆盖。页码校正改为本地管理后不再出现多人冲突。

## 八、API 接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/health | 健康检查，返回 template/attempt 数量 |
| GET | /api/data | 获取 templates、attempts、cursors |
| PUT | /api/data | 上传增量变更（仅推脏数据，按 key upsert） |

注：`project_config` 已从 API 中移除，不走云端。

## 十一、已知注意事项

1. 图片存在服务器 `/data/` 路径下，Nginx 有 7 天缓存
2. 服务器停止后公网 IP 会变化（未绑弹性 IP）——重启后需更新 SSH 配置
3. 浏览器兼容：推荐 Chrome/Edge
4. 标注完模板页建议停留几秒再切到副本页
