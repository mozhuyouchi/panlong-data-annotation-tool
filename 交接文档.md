# 习题册数据标注工具 — 项目交接文档

## 一、项目概述

一个**习题册数据标注工作台**，让多位标注员在不同浏览器上同时对扫描的数学练习册进行标注（框选题干位置、标记答案、判断对错），数据通过后端 API 实时共享。

| 项目项 | 信息 |
|--------|------|
| GitHub 仓库 | https://github.com/mozhuyouchi/panlong-data-annotation-tool |
| 在线地址 | http://47.98.108.88/ |
| 服务器 | 阿里云 ECS（Ubuntu，1.6G 内存，40G 磁盘） |

## 二、技术架构

```
浏览器 ──→ Nginx (端口80)
              ├── /           → 静态网页 (/var/www/html/)
              ├── /data/      → 扫描图片 (/var/www/html/data/, 22GB)
              └── /api/*      → Flask API (127.0.0.1:5000) → SQLite
```

| 层 | 技术 | 说明 |
|----|------|------|
| 前端 | 纯 HTML/CSS/JS | 单页面应用，无框架 |
| 后端 API | Python Flask | 3 个接口，约 120 行代码 |
| 数据库 | SQLite | 存标注数据，不存图片 |
| 反向代理 | Nginx | 同域名避免 CORS 问题 |

## 五、标注工作流程

### 1. 导入配置
打开 http://47.98.108.88/ → 「导入配置 JSON」→ 选择 `project-config.json`

### 2. 选择标注员和任务
- 左侧下拉选择标注员（001~013）
- 点击「进入我的任务」
- 任务列表会显示该标注员负责的书本和页范围

### 3. 开始标注
- 系统自动加载服务器上的扫描图片
- **无需从本地选择文件夹**

#### 标注流程：
1. 选择当前题目类型（单选题/多选题/填空题/问答题）
2. 在图片上**框选题干区域** → 再**框选作答区域**
3. 输入标准答案、学生答案、得分、状态（正确/错误）
4. 点「标记本页完成」→ 自动保存到服务器

### 4. 模板与副本
- 每本书有一个**模板书**（如「0-朱润南」），由先标注的人创建模板
- 其他副本（学生）自动复用模板的题干框位置
- 模板新增题目后，其他副本刷新页面会自动追加

### 5. 页码校对
- 如果扫描图与逻辑页不匹配，在右侧「页码对齐」面板手动输入正确的逻辑页码
- 点击「确认并重排后续页」修正

### 6. 导出结果
- 完成标注后，点击「导出任务 JSON」
- 导出内容包含：题目框坐标、标准答案、学生答案、得分状态、图片路径

## 十、已知注意事项

1. 图片存在服务器 `/data/` 路径下，Nginx 有 7 天缓存
2. 本地文件夹模块已隐藏（保留代码），如需恢复删掉 HTML 中的 `hidden` 类即可
3. 服务器停止后公网 IP 会变化（未绑弹性 IP）——重启后需更新 SSH 配置和访问地址
4. 数据同步每 5 秒推送一次，其他浏览器需刷新才能看到新数据
5. 浏览器兼容：推荐 Chrome/Edge
